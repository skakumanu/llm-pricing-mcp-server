Client Setup
Recommended. No installation required — your client connects directly to the hosted server at https://llm-pricing-api.fly.dev/mcp.
1

Add to your client's MCP config

Use the url field — no command or local path needed.

{ "mcpServers": { "llm-pricing": { "url": "https://llm-pricing-api.fly.dev/mcp" } } }
2

Restart your client

After saving the config, restart Claude Desktop / VS Code / Cursor. The 15 tools will appear automatically.

3

Ask a question

Try: "What is the cheapest model for RAG under $2/1M tokens?"

Option A (remote, easiest): use the Remote HTTP tab above. Option B below runs a local Python server — works offline.
1

Clone and install

git clone https://github.com/skakumanu/llm-pricing-mcp-server.git cd llm-pricing-mcp-server python -m venv .venv # macOS/Linux: source .venv/bin/activate # Windows: .venv\Scripts\Activate.ps1 pip install -r requirements.txt
2

Edit claude_desktop_config.json

Windows: %APPDATA%\Claude\claude_desktop_config.json macOS: ~/Library/Application Support/Claude/claude_desktop_config.json
{ "mcpServers": { "llm-pricing": { "command": "python", "args": ["-u", "/absolute/path/to/llm-pricing-mcp-server/mcp/server.py"], "cwd": "/absolute/path/to/llm-pricing-mcp-server", "env": { "PYTHONUNBUFFERED": "1" } } } }
3

Restart Claude Desktop

Tools appear in Claude's tool list. You'll see a hammer icon when the MCP server is active.

Works with: Cline, Continue, GitHub Copilot (agent mode), and any VS Code extension that supports MCP servers.
1

Open your MCP config

In Cline: Settings (⚙) → MCP Servers → Edit JSON. For .vscode/mcp.json in your workspace, create the file if it doesn't exist.

2

Add the server

{ "mcpServers": { "llm-pricing": { "url": "https://llm-pricing-api.fly.dev/mcp" } } }

Reload the extension

Open the command palette (Ctrl+Shift+P) → Cline: Reload MCP Servers. Tools will appear in the agent panel.

Restart Perplexity

Troubleshooting
ProblemFix
Tools not showing after restartCheck the config file path and JSON syntax. Use a JSON validator if unsure. Ensure no trailing commas.
STDIO server hangs with no outputExpected — the STDIO server waits for JSON-RPC input on stdin. It is not a web server; use it only with a MCP client.
ModuleNotFoundError (local STDIO)Activate the virtualenv (source .venv/bin/activate) and run from the repo root directory.
Model not found errorCall get_all_pricing first to get valid model names — they are case-sensitive.
Connection refused / timeoutRun curl https://llm-pricing-api.fly.dev/health to verify the server is up. If using local STDIO, ensure the server process is running.
401 Unauthorized on MCP endpointThe /mcp endpoint is public — no API key needed. If you see 401, check for a misconfigured auth header in your client.
